# 카피바라 Go! - 프로젝트 상세

## 개요
카피바라 Go 스타일 텍스트 기반 로그라이크 RPG 웹게임
- 플랫폼: 웹 브라우저 (모바일 우선)
- 언어: 한국어
- 개발: 5명 에이전트 팀 (2026-02-09)

## 파일 구조
```
09-capybara-go/
├── index.html          (155줄) - HTML 구조
├── ARCHITECTURE.md     - 아키텍처 설계서
├── CAPYBARA_GO.md      - 이 파일
├── css/
│   └── style.css       (1174줄) - 다크 테마 스타일링
├── js/
│   ├── data.js         (1307줄) - 게임 데이터 (스킬/적/장비/텍스트)
│   ├── mechanics.js    (625줄) - 전투/이벤트 로직
│   └── game.js         (877줄) - 메인 게임 엔진
└── assets/             - 이미지 에셋 (미사용, 이모지 기반)
```

## 게임 시스템
- **챕터**: 3개 (숲→사막→화산), 각 60일
- **전투**: 자동 전투, 최대 20라운드
- **스킬**: 32개 (일반 20, 전설 7, 신화 5)
- **인카운터**: 16개 이벤트 (선택/함정/행운)
- **적**: 15종 (챕터별 4 일반 + 1 보스)
- **장비**: 12개 (무기/방어구/장신구 각 4)
- **펫**: 6종 (전투 보조)
- **저장**: localStorage 기반

## 기술 스택
- 순수 HTML/CSS/JS (프레임워크 없음)
- Google Fonts (Noto Sans KR)
- CSS 애니메이션 (float, shake, fadeIn, slideUp, glow 등)
- 모바일 반응형 (max-width: 480px)

## 버그 수정 이력
- **전투 결과 안 보임** (늑대 전투 후 프리즈): CSS `.active` 클래스 vs JS `style.display` 불일치 → `classList.add/remove('active')` 통일
- **인벤토리 안 열림**: 동일 원인 → `classList` 방식으로 수정
- **로그 자동 스크롤 안 됨**: JS에서 `.slide-up` 사용했으나 CSS에는 `.log-entry` 정의 → 클래스명 통일 + `requestAnimationFrame()` 적용
- 3개 버그 모두 멀티 에이전트 개발 시 CSS↔JS 인터페이스 불일치가 원인
